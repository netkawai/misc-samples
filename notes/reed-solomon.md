リードソロモン符号計算法
リードソロモンとはブロック単位の誤り符号である。
2進数の3bitを1ブロックとしたデータとして、(1,7,3)を送るとする。
このデータが誤っているどうか単純に知る方法は、ある計算を行って
例えばゼロになるような値を一緒に送ればいい。
(1,7,3) = (W1,W2,W3)としたとき、これに付加する誤り訂正符号をWcとすると
W1 + W2 + W3 + Wc = 0になる値を求める。ここで、ここでの計算は、2^3 = 8を
法とした計算とし、計算方法としては通常の10進計算をした後、
8で割ったあまりの数とする。
Wc = W1 + W2 + W3 = 1 + 7 + 3 = 5
(W1,W2,W3,Wc) = (1,7,3,5)で1 + 7 + 3 + 5 = 0 になる
受信したデータが(1,4,3,5)の場合1 + 4 + 3 + 5 = 5 ≠ 0 なので、
誤りを含むことが分かる。
これよりさらに、誤りを訂正できる用にしようとすれば、
W1 +   W2 +   W3 + Wc1 = 0
W1 + 2*W2 + 3*W3 + Wc2 = 0
になる2つの符号を求める。
この場合、Wc1 = 5, Wc2 = 0になる
1 + 7 + 3 + 5    = 16 = 0 mod 8
1 +2*7 + 3*3 + 0 = 24 = 0 mod 8
(1,7,3,5,0)になる。
これを受信したデータが(1,7,6,5,6)になった場合
誤り値は 1 + 7 + 6 + 5 = 19 = 3 mod 8
誤り位置は 1 + 2*7 + 3*6 + 0 = 1で
1/5 = 1*5^-1 = 1*3 = 3で、3を掛ける位置に3引いた値であるので、
(1,7,6,5,0)を(1,7,3,5,0)に戻すことが出来る。
